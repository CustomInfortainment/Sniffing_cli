# Canshell

차량 CAN 버스 데이터를 수집하는 CLI 툴 입니다.

---

## 개요

차량 인포테인먼트 시스템 개발 과정에서 CAN 데이터 스니핑 코드의 테스트 편의성을 높이기 위해 제작되었습니다.  
빌드 후 본 프로그램에서 제공하는 옵션을 통해 데이터 저장, 출력 등을 유연하게 제어할 수 있습니다.

---

## 개발 환경

- **OS**: Mac OS 터미널 환경
- **언어**: C
- **하드웨어**: CAN-Serial 변환 모듈 (Canable 2.0 활용)

---

## 사용법

본 툴은 각 명령 기능 구현 및 데이터 수신 기능 분리를 위한 모드들이 존재합니다.

COMMAND모드 : 사용자가 차량으로부터 CAN프레임 데이터를 수집하기 전, 설정을 위한 명령을 내리는 모드입니다.<br>
RUNNING모드 : 차량으로부터 CAN 프레임 데이터를 수신하고 있는 상태입니다.

모드 전환 방법은 다음과 같습니다 :

COMMAND -> RUNNING : 명령어 'r' 혹은 "run"을 입력한다.<br>
RUNNING -> COMMAND : 명령어 'q' 혹은 "quit" 를 입력한다.

## 명령어 모음

| 명령어 | 사용 모드 | 설명 |
|------|-----|-----|
| filter [id] | COMMAND | 원하는 ID만 출력 및 저장할 수 있습니다. |
| mask [id] | COMMAND | 해당 ID를 제외한 모든 ID를 출력합니다. |
| unfilter | COMMAND | 필터링으로 지정한 ID를 모두 초기화 합니다. (ID와 같이 입력시 특정 ID만 필터링 해제도 가능합니다.)
| unmask | COMMAND | 마스크로 지정한 ID를 모두 초기화 합니다. (ID와 같이 입력시 특정 ID만 마스크 해제도 가능합니다.)
| run | COMMAND | RUNNING 모드로 전환합니다. (CAN통신을 시작합니다.)
| quilt | RUNNING | COMMAND 모드로 전환합니다. (CAN통신을 중지합니다.)

## 명령 옵션
⚠️ 본 옵션은 run 명령 실행 시 인자로 들어갑니다.

| 명령어 | 설명 |
|------|-----|
| -s | 수집한 데이터를 저장합니다. |
| -p | 수집한 데이터를 실시간으로 출력합니다. |

### 명령어 활용 예시

```bash

sniffer > filter 0x1C9
sniffer > run -s -p

RUNNING 모드로 전환...

running > id:0F9 DLC:8, data:00 00 40 00 00 00 00 00
running > id:0F9 DLC:8, data:00 00 40 00 00 00 00 00
running > id:0F9 DLC:8, data:00 00 40 00 00 00 00 00
running > id:0F9 DLC:8, data:00 00 40 00 00 00 00 00

q 혹은 quit 입력시

CAN통신을 중지합니다...

sniffer >

```

## 프로젝트 배경

원래 차량 인포테인먼트 시스템 개발의 일부로 시작되었으나, 스니핑 코드 테스트 시 매번 코드를 수정해야 하는 불편함이 있어 CLI 툴로 분리하였습니다.

### 기존 문제점

- 데이터 저장 여부, 출력 여부를 변경하려면 코드 주석 처리 후 재빌드 필요
- 수많은 데이터 파싱 시 매번 코드를 수정해야 하는 번거로움 존재

### 해결 방향

- CLI툴로 수정, 코드 수정 없이 실시간으로 바로 데이터를 파싱할 수 있게 되었음.
